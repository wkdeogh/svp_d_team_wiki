# SVP D팀 위키 웹앱 구현 계획

## 1. 목표
- 삼성전자 입사 동기 모임인 SVP D팀의 위키형 웹앱을 만든다.
- 팀의 기록, 추억, 행사, 개인 소식, 방명록을 한 곳에 모은다.
- PC와 모바일에서 모두 자연스럽게 사용되도록 반응형으로 설계한다.
- Supabase를 백엔드로 사용하고, Vercel에 배포한다.

## 2. 핵심 기능

### 2.1 방명록
- 누구나 글을 남길 수 있는 자유 게시판.
- 필드: 이름 또는 닉네임, 내용, 작성일.
- 최신순 기본 정렬.
- 스팸 방지를 위해 간단한 제한을 둔다.
- 선택 기능: 이모지 반응, 삭제 요청.

### 2.2 D팀 역사 타임라인
- 이 프로젝트의 핵심 기능.
- 누구나 제목, 내용, 날짜를 추가할 수 있다.
- 예시: 결혼, 행사, 회식, 여행, 인사, 프로젝트 완료 등.
- 날짜 오름차순 정렬로 시간 흐름이 보이게 한다.
- 필드: 제목, 내용, 날짜, 카테고리, 작성자, 첨부 이미지(선택).
- 필터: 연도, 카테고리, 사람 이름 검색.

### 2.3 사진 아카이브
- 행사, 모임, 회식, 여행 사진을 모아볼 수 있는 공간.
- 앨범 단위 정리 가능하게 설계한다.
- 사진 업로드 시 캡션과 날짜를 함께 저장한다.
- 타임라인 항목과 연결할 수 있으면 더 좋다.

### 2.4 멤버 소개
- D팀 구성원을 소개하는 페이지.
- 필드: 이름, 닉네임, 한줄 소개, 입사 동기, 관심사, 대표 사진.
- 연락처나 민감 정보는 저장하지 않는다.
- 각 멤버별 추억 글이나 타임라인 항목으로 연결 가능하게 한다.

### 2.5 검색 및 필터
- 전체 콘텐츠에서 검색 가능하게 한다.
- 필터 대상: 날짜, 카테고리, 작성자, 이름.
- 모바일에서도 쉽게 조작할 수 있는 칩 형태 필터를 사용한다.

### 2.6 반응/좋아요
- 게시물과 사진에 좋아요 또는 공감 버튼을 둔다.
- 인기 있는 추억을 상단에 노출할 수 있다.

### 2.7 관리자 기능
- 수정/삭제/숨김 처리.
- 신고된 콘텐츠 검토.
- 승인 후 공개 옵션도 고려한다.

## 3. 추천 추가 기능
- 연말 결산 페이지
- 기념일 캘린더
- 인기 게시물 랭킹
- 태그 기능
- 무작위 추억 카드
- 모바일 홈 위젯 느낌의 요약 카드

## 4. 정보 구조

### 페이지 구성
- `/` 홈
- `/guestbook` 방명록
- `/history` D팀 역사 타임라인
- `/photos` 사진 아카이브
- `/members` 멤버 소개
- `/calendar` 기념일 캘린더
- `/admin` 관리 페이지

### 홈 구성
- 최근 추가된 역사 5개
- 최근 방명록 5개
- 이번 달 행사/기념일
- 사진 하이라이트
- 빠른 등록 버튼

## 6. Supabase 설계

### 6.1 인증
- 초기 버전은 로그인 없이 작성 가능하게 둘 수 있다.
- 다만 관리자 기능은 Supabase Auth로 보호한다.
- 추후 멤버만 수정 가능하도록 확장할 수 있게 설계한다.

### 6.2 Row Level Security
- 공개 읽기 허용.
- 작성은 제한된 정책으로 허용.
- 수정/삭제는 작성자 또는 관리자만 가능하게 한다.
- 민감 데이터는 저장하지 않는다.

### 6.3 Storage
- `photos` 버킷 생성.
- 프로필 이미지와 행사 사진 업로드 분리 가능.
- 썸네일용 이미지는 선택적으로 생성한다.

### 6.4 트리거/정책
- 생성 시 `created_at` 자동 기록.
- 검색용 `tsvector` 또는 인덱스 추가 고려.
- 타임라인 날짜 정렬 인덱스 추가.

## 7. UI/UX 원칙

### 공통 원칙
- 모바일 우선 설계.
- 카드 기반 레이아웃 사용.
- 입력 폼은 짧고 단순하게 유지.
- 주요 액션은 상단과 하단에 반복 배치.
- 빈 상태 화면을 친근하게 만든다.

### PC 화면
- 좌측 사이드 내비게이션 또는 상단 탭.
- 넓은 타임라인 영역과 보조 패널 분리.
- 사진은 그리드로 크게 보여준다.

### 모바일 화면
- 하단 고정 또는 상단 접이식 내비게이션.
- 한 화면에 한 작업만 집중되게 구성.
- 입력 버튼은 엄지로 누르기 쉬운 크기로 둔다.

- 환경변수 설정.

## 9. 기술 스택 제안
- Frontend: Next.js
- Styling: Tailwind CSS 또는 CSS Modules
- Backend: Supabase
- DB: PostgreSQL
- Auth: Supabase Auth
- Storage: Supabase Storage
- Deploy: Vercel

## 10. 운영 정책
- 공개 작성 기능은 스팸 방지 장치를 둔다.
- 관리자가 언제든 숨기기/삭제 가능해야 한다.
- 민감 정보는 등록하지 않도록 안내 문구를 둔다.
- 백업과 복구를 정기적으로 확인한다.

## 11. 완료 기준
- 방명록이 등록/조회된다.
- 역사 타임라인이 날짜 오름차순으로 보인다.
- 누구나 역사 항목을 추가할 수 있다.
- 사진과 멤버 소개 페이지가 동작한다.
- 모바일과 PC 모두에서 레이아웃이 깨지지 않는다.
- Vercel 배포가 완료된다.
